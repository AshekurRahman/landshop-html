Installation

Follow the steps below to setup your site template:

  1. Unzip the downloaded package and open the /HTML folder to find all the template files. You will need to upload these files to your hosting web server using FTP or localhost in order to use it on your website.
  2. Below is the folder structure and needs to be uploaded to your website or localhost root directory:
    • HTML/assets - Contains all of the assets referenced
      • HTML/assets/css - Stylesheet files
      • HTML/assets/fonts - All fonts and icons files.
      • HTML/assets/images - Images files
      • HTML/assets/js - Javacript files
    • HTML/style.css - This Stylesheet to write your own code.
    • HTML/index.html - Backpack Landing Page
    • HTML/index-2.html - VR Landing Page
    • HTML/index-3.html - Speakers Landing Page
    • HTML/index-4.html - Cycle Landing Page
    • HTML/index-5.html - Webcam Landing Page
    • HTML/index-6.html - Drone Landing Page
    • HTML/index-7.html - Skateboard Landing Page
    • HTML/index-8.html - Sunglass Landing Page
    • HTML/index-9.html - Airpods Landing Page
    • HTML/404.html - 404 error page
    • HTML/about.html - About Page
    • HTML/blog.html - Blog List Page
    • HTML/blog-details.html - Blog Details page
    • HTML/contact.html - Dynamic Contact Page
    • HTML/faq.html - Frequently Asked Questions Page
    • HTML/products.html - Product List Page
    • HTML/product-details.html - Product Details Page
    • HTML/process.php - This is for Contact form
  3. You should upload all or specific HTML files as per your need.
  4. You are good to go for adding your content now!

CSS

The Landshop template includes a number of CSS files in the HTML/assets/css directory that you can use to customize the CSS styles for your website. If you have experience working with CSS, you can modify these files directly and customize the styles to your liking.

We have provided two different homepage options, each with its own corresponding CSS file: theme-1.css for Homepage 1 and theme-2.css for Homepage 2. These files include styles specific to each homepage, such as layout, typography, and color schemes.

Additionally, we have included a responsive.css file that contains CSS styles for ensuring that the website is responsive and looks good on different devices and screen sizes. You can modify this file to adjust the styles and layout for different breakpoints.

For Homepage 1, we have also included a responsive-1.css file that contains specific CSS styles for ensuring that the homepage 1 is responsive and looks good on different devices and screen sizes.

In addition, we have also included a style.css file where you can add your own custom CSS styles. This file is where you can add your own styles to modify or extend the existing styles of the template.

To get started with CSS customization, open the relevant CSS file(s) in your preferred code editor and make your modifications. You can use any text editor or integrated development environment (IDE) that supports CSS, such as Visual Studio Code, Sublime Text, or Atom.

Once you have made your customizations, save the file(s) and refresh your website in the browser to see the changes. If you need more information on specific styles or elements, you can refer to the comments and documentation within the CSS files.

Code

Documentation and examples for displaying inline and multiline blocks of code

Inline code

Wrap inline snippets of code with <code>. Be sure to escape HTML angle brackets.

For example, <section> should be wrapped as inline.

For example, <code>&lt;section&gt;</code> should be wrapped as inline.

Code blocks

Use <pre>s for multiple lines of code. Once again, be sure to escape any angle brackets in the code for proper rendering. You may optionally add the .pre-scrollable class, which will set a max-height of 340px and provide a y-axis scrollbar.

For example:

<p>Sample text here...</p>
<p>And another line of sample text here...</p>

Have to use something like:

<pre><code>&lt;p&gt;Sample text here...&lt;/p&gt;
&lt;p&gt;And another line of sample text here...&lt;/p&gt;
</pre></code>

PHP Codes

  <?php
    // Configure your Subject Prefix and Recipient here
    $subjectPrefix = 'Landshop';
    $emailTo       = 'yourmail@example.com';
    $errors = array(); // array to hold validation errors
    $data   = array(); // array to pass back data
    if($_SERVER['REQUEST_METHOD'] === 'POST') {
        $name    = stripslashes(trim($_POST['name']));
        $email   = stripslashes(trim($_POST['email']));
        $subject = stripslashes(trim($_POST['subject']));
        $date = stripslashes(trim($_POST['date']));
        $message = stripslashes(trim($_POST['message']));
        if (empty($name)) {
            $errors['name'] = 'Name is required.';
        }
        if (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
            $errors['email'] = 'Email is invalid.';
        }
        if (empty($subject)) {
            $errors['subject'] = 'Subject is required.';
        }
        if (empty($date)) {
            $date = date("Y/m/d");
        }
        if (empty($message)) {
            $errors['message'] = 'Message is required.';
        }
        // if there are any errors in our errors array, return a success boolean or false
        if (!empty($errors)) {
            $data['success'] = false;
            $data['errors']  = $errors;
        } else {
            $subject = "$subjectPrefix $subject";
            $body    = '
                <strong>Name: </strong>'.$name.'<br />
                <strong>Email: </strong>'.$email.'<br />
                <strong>Date: </strong>'.$date.'<br />
                <strong>Message: </strong>'.nl2br($message).'<br />
            ';
            $headers  = "MIME-Version: 1.1" . PHP_EOL;
            $headers .= "Content-type: text/html; charset=utf-8" . PHP_EOL;
            $headers .= "Content-Transfer-Encoding: 8bit" . PHP_EOL;
            $headers .= "Date: " . date('r', $_SERVER['REQUEST_TIME']) . PHP_EOL;
            $headers .= "Message-ID: <" . $_SERVER['REQUEST_TIME'] . md5($_SERVER['REQUEST_TIME']) . '@' . $_SERVER['SERVER_NAME'] . '>' . PHP_EOL;
            $headers .= "From: " . "=?UTF-8?B?".base64_encode($name)."?=" . "<$email>" . PHP_EOL;
            $headers .= "Return-Path: $emailTo" . PHP_EOL;
            $headers .= "Reply-To: $email" . PHP_EOL;
            $headers .= "X-Mailer: PHP/". phpversion() . PHP_EOL;
            $headers .= "X-Originating-IP: " . $_SERVER['SERVER_ADDR'] . PHP_EOL;
            mail($emailTo, "=?utf-8?B?" . base64_encode($subject) . "?=", $body, $headers);
            $data['success'] = true;
            $data['message'] = 'Congratulations. Your message has been sent successfully';
        }
        // return all our data to an AJAX call
        echo json_encode($data);
    }

Subscribe Form

To set up a Mailchimp subscribe form on your HTML template, you can follow these steps:

  1. Create a Mailchimp account and log in.
  2. In the Mailchimp dashboard, navigate to the "Audience" tab and select "Audience dashboard".
  3. Click on "Signup forms" and then "Embedded forms".
  4. Customize your form settings and fields as per your requirements.
  5. Click on the "Copy" button to copy the HTML code for your form.
  6. Open the HTML file where you want to add the subscribe form in any text editor.
  7. Paste the HTML code where you want the form to appear on the page.
  8. Make sure to save the HTML file and upload it to your server.

Once you have added the Mailchimp subscribe form to your HTML template, your website visitors can use the form to subscribe to your mailing list. The email addresses collected through the form will be added to your Mailchimp audience automatically.

FAQ

A FAQ is a list of frequently asked questions (FAQs) and answers on a particular topic.

What is an HTML template?
An HTML template is a pre-designed and pre-coded website or web application that can be customized and used as the basis for your own project. HTML templates can save time and effort in the development process and provide a professional and polished look to your website or web application.
Do I need to know how to code to use an HTML template?
Basic knowledge of HTML, CSS, and JavaScript is required to customize an HTML template. However, many templates come with detailed documentation and support to help users customize and implement the template.
What do I get when I purchase an HTML template?
When you purchase an HTML template, you typically receive the HTML, CSS, and JavaScript files that make up the template. You may also receive additional assets like images, fonts, and icons that are used in the template.
Can I use an HTML template for commercial projects?
Yes, most HTML templates can be used for commercial projects, but it is important to check the license agreement before using the template to ensure that you are following the terms and conditions.
How do I customize an HTML template?
You can customize an HTML template by editing the HTML, CSS, and JavaScript files. Most templates come with documentation that explains how to make changes and customize the template to fit your needs.
What if I have problems customizing the template?
Most HTML template providers offer customer support and documentation to help users customize and implement the template. If you have any problems, you can reach out to the provider for help.
Are HTML templates mobile-friendly?
Most modern HTML templates are designed to be mobile-friendly and responsive, meaning that they will look and function well on a variety of devices, including desktops, laptops, tablets, and smartphones.
Can I use a HTML template with a content management system (CMS)?
Yes, you can usually use HTML templates with CMS platforms like WordPress, Drupal, and Joomla. However, it is important to check compatibility and ensure that the template is designed to work with your chosen CMS.
